Developing Windows Xtras

The compiler currently supported for developing Xtras in Windows 95, 98 and NT is Microsoft Visual C++ version 6.0.

To build a new Xtra, open one of the example .dsw files in Microsoft Visual C++ (e.g. XDK\Examples\Sprite\Oval\WinProj\Oval.dsw). Choose Rebuild All from the Build menu. In Win32, the compiler will create a Oval.x32 shared library in the Oval\Release directory. If you set the project to make a debug version, a debug version of the compiled Xtra will be put in Oval\Debug.

Debugging Windows Xtras

Steps

  1. Open VC and build the Debug target for the project, as described above (i.e. the Oval project).

  2. Copy the Debug version of the Xtra (i.e. winproj\Debug\Oval.x32) or a shortcut into the Director 8\Xtras folder.
  3. Switch back to VC, click "Project" and then "Settings".
  4. Select the "Debug" tab. Type in the full path to Director 8 in the "Executable for debug session" text field. Or use the right arrow button to browse to it.
  5. Set breakpoints in one of your source files.
  6. Click on the "Build" menu item, select "Start Debug" and then "Go". Or use the F5 shortcut.
  7. Click "OK" when it warns you that Director 8 contains no debugging information.
