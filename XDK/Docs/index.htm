Overview of what's new in the D8 XDK

Director Xtra Development Kit

Introduction

This release of the Director Xtra Development Kit is for use in developing Xtras for Director 8.0. The documentation for the additions and changes made to the Director 8 XDK are in the xdk8_addendum.html file in the Docs folder.

What's an Xtra?

Xtras are code extensions to Macromedia applications such as Director and Authorware. For the user, Xtras add custom features to tools they already know how to use. For the developer, Xtras are C-code object modules that use the Xtra Application Programming Interface (API) to extend the functionality of Macromedia applications. All Xtra API is implemented using the Macromedia Open Architecture (MOA). MOA defines an object model that provides standard ways for applications and extensions to interact.

An Xtra Development Kit (XDK) provides the resources you need to implement Xtras for Macromedia products. In addition to documentation, this XDK provides header files for the API supported by Director, and examples of how to implement several kinds of Director Xtra.

The resources provided by the Director XDK are organized in the following directories:

Directory

Contents

Docs

Director XDK and MOA documentation in html format

Examples

Project files for Director Xtra examples

Includes

Header files for Xtra development

Utility

Utility programs for use in developing Xtras

Xtras

Compiled versions of the example Xtras

The Xtras folder should be moved to the folder where the Director application resides. The Examples and Includes folders should be placed together in a folder so that projects will build successfully.

If you haven't done so already, you should read the readme file that came with the Director XDK. It contains descriptions of the example files, updated information about API changes, known bugs, and other specific details about this version of the XDK.

 

In the documentation

The XDK documentation includes conceptual discussions of the types of Xtras you can create and complete documentation of the API of the Director XDK. The documentation consists of the following parts.

  • The Director Developer's Guide describes the API you use to implement Xtras with Director-specific behavior, as well as the support API provided specifically by Director. It also provides reference documentation for Director interfaces and types.

  • The Multimedia Developer's Guide describes the API you use to implement a multimedia Xtra, as well as the support API provided by Director. It also provides reference documentation for multimedia interfaces and types.

  • The MOA File Access II Developer's Guide documents the File Access II API, which provides local and remote file access.

  • The MUI Developer's Guide documents a cross-platform API for displaying and interacting with dialog boxes conforming to the Macromedia User Interface standard.

  • The MOA Developer's Guide describes the Macromedia Open Architecture object model, the general structure of MOA Xtras, and the basics of the MOA Xtra development process.

  • XDK Addendum for Director 8 describes the new features and interfaces for the Director 8 release.

Note on type conventions

The following type conventions are used throughout this document:

Italic type is used to identify new terms where they are defined.

Courier type is used to identify API literals, including class names, method names, function names, type names, macros, and code fragments.

Italic courier type is used to identify parameters to functions and methods.
